# crop_path.json Schema

The crop path is the inspectable intermediate format. Generated by the `analyze` command, it contains per-keyframe crop coordinates. Users can edit this file before rendering to manually override crop positions.

## Full Schema

```json
{
  "version": "1.0",
  "source": {
    "file": "path/to/video.mp4",
    "width": 1920,
    "height": 1080,
    "fps": 30.0,
    "total_frames": 2280
  },
  "output": {
    "format": "9x16",
    "crop_w": 607,
    "crop_h": 1080
  },
  "config_used": "tools/clip-extractor/config.yaml",
  "detection_stats": {
    "frames_sampled": 380,
    "faces_detected": 372,
    "face_detected_pct": 97.9,
    "avg_confidence": 0.89,
    "sampling_rate": 6,
    "interpolated_frames": 1900
  },
  "keyframes": [
    {
      "frame": 0,
      "time_sec": 0.0,
      "crop_x": 656,
      "crop_y": 0,
      "crop_w": 607,
      "crop_h": 1080,
      "face_detected": true,
      "face_confidence": 0.92,
      "face_center_x": 0.5012,
      "face_center_y": 0.3521,
      "smoothed": true,
      "deadzone_active": false
    }
  ]
}
```

## Key Fields

| Field | Type | Description |
|-------|------|-------------|
| `keyframes[].frame` | int | Frame index in source video |
| `keyframes[].crop_x` | int | Left edge of crop window (pixels) — **editable** |
| `keyframes[].crop_w` | int | Crop width (pixels) — usually constant |
| `keyframes[].face_detected` | bool | Whether a face was found on this frame |
| `keyframes[].deadzone_active` | bool | Whether deadzone suppressed movement |
| `detection_stats.face_detected_pct` | float | Should be >90% for talking head videos |

## Manual Editing

To override the crop position for specific moments (e.g., when speaker points at a screen share):

1. Find the keyframe by `time_sec`
2. Change `crop_x` to shift the crop window left/right
3. Re-render: `python -m clip_extractor render --video FILE --crop-path crop_path.json --output FILE`

The renderer linearly interpolates between keyframes, so editing a few keyframes creates smooth transitions to the new position.

## Phase 6 Fields (Dynamic Podcast Layout)

When `--format split` detects mixed layouts (split-screen + close-up), these fields are added:

### `layout_segments` (top-level array)

```json
{
  "layout_segments": [
    {
      "layout": "split_screen",
      "start_frame": 0,
      "end_frame": 405,
      "start_sec": 0.0,
      "end_sec": 13.5
    },
    {
      "layout": "close_up",
      "start_frame": 408,
      "end_frame": 1626,
      "start_sec": 13.6,
      "end_sec": 54.2
    }
  ]
}
```

| Field | Type | Description |
|-------|------|-------------|
| `layout_segments[].layout` | string | `"split_screen"` or `"close_up"` |
| `layout_segments[].start_frame` | int | First frame of this segment |
| `layout_segments[].end_frame` | int | Last frame of this segment |
| `layout_segments[].start_sec` | float | Start time in seconds |
| `layout_segments[].end_sec` | float | End time in seconds |

### `layout_type` (per keyframe)

```json
{
  "keyframes": [
    {
      "frame": 0,
      "layout_type": "split_screen",
      ...other fields...
    }
  ]
}
```

Empty string (`""`) for backward compatibility with older crop paths.
